KT MRT-1 Freeze->Run->Fix Runbook

Assumptions
- Use PINNED_SHA unless intentionally overridden.
- During debugging, only edit stage [5.1] plan resolution.
- Autogen plans must be written under POLICY_C_EXPORT_REL (never into tracked source).
- OUT_DIR is immutable evidence for receipts and artifacts.

Quick commands
- Run full cell: paste `cell.sh` into a Kaggle `%%bash` cell and execute.
- Tail transcript:
  tail -n 300 OUT_DIR/cell_transcript.log
- Resolver-only validator discovery:
  python - <<'PY'
import policy_c.sweep_runner as sr
print([n for n in dir(sr) if 'validate' in n or 'assert' in n or 'plan' in n])
PY

Freeze->Run->Fix protocol
1. Freeze: stop editing all stages except [5.1].
2. Run: execute the full cell end-to-end.
3. Triage: map failure to stage header `[N]` in transcript.
4. Fix: make one minimal change in that stage only.
5. Re-run: execute again from [0], repeat until full pass.
6. Lock: after one full pass, set POLICY_C_PLAN_REL to the resolved plan path and keep tests committed.

Top 5 triage cases
1. FAIL_CLOSED: no sweep plan validator found
   - Discover validator names:
     python - <<'PY'
import policy_c.sweep_runner as sr
print([n for n in dir(sr) if 'validate' in n or 'assert' in n or 'plan' in n])
PY
   - Add discovered callable to the [5.1] candidate list.
2. FAIL_CLOSED: multiple valid sweep plans found (ambiguous)
   - Set POLICY_C_PLAN_REL to a single target plan and rerun.
3. NO_VALID_PLAN_FOUND followed by autogen failure
   - Inspect schema:
     python - <<'PY'
import json, pathlib
p = pathlib.Path('KT_PROD_CLEANROOM/policy_c/policy_c_sweep_plan_schema_v1.json')
print('exists', p.exists())
if p.exists():
    s = json.loads(p.read_text(encoding='utf-8'))
    print('top keys', list(s.keys())[:50])
PY
   - Adjust autogen or use example override plan.
4. FAIL_CLOSED: no dataset files produced by sweep
   - Inspect export directory:
     ls -la exports/policy_c/<RUN_ID>/sweep_a
     find exports/policy_c/<RUN_ID>/sweep_a -type f -name '*.jsonl*' -ls
5. FAIL_CLOSED: training failed for <adapter>
   - Inspect adapter log:
     tail -n 200 OUT_DIR/mrt1_train_stdout/<adapter>.log

Proof checkpoints
- After [5.1]: OUT_DIR/policy_c_sweep_plan_resolution_receipt.json exists.
- After [5.2]: OUT_DIR/dataset_raw.jsonl exists and is non-empty.
- After [6]: OUT_DIR/dataset_coercion_report.json contains "gate_D2": "PASS".
- After [7]: OUT_DIR/mrt1_adapter_ids.txt has exactly 13 lines.
- After [8]: OUT_DIR/mrt1_promotion_receipts/<adapter>_train_receipt.json exists for each trained adapter.
- Final: OUT_DIR.tar.gz and OUT_DIR.tar.gz.sha256 exist and hash matches.
