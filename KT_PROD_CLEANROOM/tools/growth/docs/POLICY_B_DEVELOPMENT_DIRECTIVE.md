# POLICY B DEVELOPMENT DIRECTIVE

Scope
- All work occurs under `KT_PROD_CLEANROOM/`.
- No side repos, no nested experiments, no ad-hoc scripts.

Reference rules
- Review V1 Temple and archived audits before modifying wiring.
- Do not copy code forward blindly.
- Do not modify Temple or archives.

Authorization
- Allowed: wiring, binding, schema enforcement, artifact emission.
- Forbidden: curriculum tuning, adapter retraining escalation, pressure increases.

Fail-closed law
- If wiring is incomplete, execution halts.
- Missing artifacts are not stubbed.
- Unknown inputs are not coerced silently.
