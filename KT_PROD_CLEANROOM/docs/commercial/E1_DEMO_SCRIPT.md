# KT E1 Demo Script

Use this script when demonstrating KT inside the bounded `E1` trust wedge.

## Demo Promise

KT does not ask the viewer to trust a slogan. KT shows a governed run, a proof packet, and a verifier path that reproduces the bounded claim on the same host.

## Demo Flow

1. State the bounded problem.
   "KT is operating under `E1_SAME_HOST_DETACHED_REPLAY`. This demo proves replayable, verifier-backed execution on the same host. It does not prove cross-host reproducibility."

2. Show the runtime step.
   Run the bounded KT flow and show the emitted packet or receipt.

3. Show the evidence packet.
   Point to the packet, the verifier kit, and the audit manifest.

4. Hand over the verifier command.
   Use the verifier command exactly as packaged, without improvising stronger claims.

5. Run verification.
   Show the bounded PASS or FAIL surface and the receipt that was produced.

6. State the boundary again.
   "This proves governed execution, replay, and bounded runtime trust at `E1`. It does not close `C006` or unlock `E2+`."

7. Offer the next step.
   Move to presales diagnostic, bounded pilot, or verifier handoff.

## What To Avoid

- Do not claim cross-host proof.
- Do not claim hostile or outsider verification.
- Do not claim superiority.
- Do not claim enterprise readiness beyond the bounded trust wedge.
